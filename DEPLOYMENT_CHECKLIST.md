# Netlify Deployment Checklist

✅ **Code is ready for production**
- Frontend build: `npm run build` ✓
- Backend serverless wrapper: `netlify/functions/api.js` ✓
- Environment-aware API URLs: ✓

## Step 1: Push to GitHub
```bash
git add .
git commit -m "Prepare for Netlify full-stack deployment"
git push origin main
```

## Step 2: Connect to Netlify
1. Go to [netlify.com](https://netlify.com)
2. Sign in (create account if needed)
3. Click **Add new site** → **Import from Git**
4. Select **GitHub** and authorize
5. Choose your repository: `procomm-eng-literary`
6. Build settings should auto-fill:
   - Build command: `npm run build`
   - Publish directory: `dist`
7. Click **Deploy site**

## Step 3: Set Environment Variables ⚠️ CRITICAL
1. Go to **Site settings** (top-right of dashboard)
2. Go to **Build & deploy** → **Environment**
3. Click **Edit variables**
4. Add these 6 variables:

```
DB_HOST = <your-cloud-mysql-host>
DB_USER = <your-mysql-username>
DB_PASSWORD = <your-mysql-password>
DB_NAME = procomm_literary
JWT_SECRET = hawkins_lab_secret_key_2024
ADMIN_PASSWORD = procommelc@2026
```

**Do NOT include `.env` file in Git** - Netlify reads environment variables from Site Settings only.

## Step 4: Set Up Cloud MySQL Database
Netlify Functions cannot access your local MySQL. Choose one:

### Option A: Railway (Recommended - Quick Setup)
1. Go to [railway.app](https://railway.app)
2. Sign up free
3. Create new project → MySQL
4. Get connection string
5. Extract: `DB_HOST`, `DB_USER`, `DB_PASSWORD`, `DB_NAME`

### Option B: PlanetScale (Free MySQL)
1. Go to [planetscale.com](https://planetscale.com)
2. Sign up free
3. Create database
4. Get MySQL credentials

### Option C: Aiven (Free Trial)
1. Go to [aiven.io](https://aiven.io)
2. Sign up free
3. Create MySQL service
4. Get credentials

## Step 5: Redeploy
After setting environment variables:
1. Netlify UI → **Deploys**
2. Click **Trigger deploy** → **Deploy site**
   
   (Or push a new commit to auto-trigger)

## Step 6: Verify Backend is Running
1. Wait for deploy to complete (green checkmark)
2. Visit your site: `https://your-site-name.netlify.app`
3. Go to Admin Login and test
4. If it fails, check:
   - Netlify deploy logs (Deploys tab)
   - Function logs (Functions → api → Logs)
   - MySQL connection credentials

## Common Issues & Fixes

### "Backend not working"
- [ ] Check Netlify deploy logs for errors
- [ ] Verify all 6 env vars are set in Site Settings
- [ ] Confirm MySQL credentials are correct
- [ ] Redeploy after changing env vars

### "Cannot connect to database"
- [ ] Use cloud MySQL, not localhost
- [ ] Test MySQL credentials locally first
- [ ] Ensure MySQL is publicly accessible

### "404 on /api/* endpoints"
- [ ] Verify `netlify.toml` exists at root
- [ ] Check Function logs for import errors
- [ ] Redeploy to refresh Function bundle

## No `.env` Files in Git

The `.gitignore` excludes all `.env` files. This is correct and secure.

Environment variables come from Netlify Site Settings, not from `.env` files.
